## 0. 最关键的前置说明

在同一套公司系统账号下，左侧菜单区分**美工版**（设计部使用）和**运营版**（深圳运营、成都运营共用）两个独立页面路由。「查看已完成生图」必须在两版里各有一个独立页面，且两版使用独立的前端页面、后端代码和数据库表，不能做成同一个页面里的美工/运营切换。

页面入口：在【新版AI生图】下增加【美工版查看已完成生图】和【运营版查看已完成生图】两个子菜单。

## 1. 背景与目标

AI 生图工作台已支持任务创建、生图、审核、推送。当任务进入「已完成」后，用户需要一个独立的只读回看页面，用于按任务和图片维度查询历史生图结果，并打开既有 L3 沉浸大屏查看单张图片。

本页面不承担创建、编辑、审核、推送等状态变更能力。

目标用户：

| 版本 / 页面 | 用户群 |
| --- | --- |
| 美工版 | 设计部专员、组主管、子部门高级主管、设计总监 |
| 运营版 | 深圳运营、成都运营的专员、组主管、部门高级主管、运营总监，以及业务管理部、总监助理、FBA/FBM 策略管理组等特殊团队成员 |
| 管理与运维角色 | 本项目产品经理、本项目前后端研发、本项目测试、技术支持 |

产品目标：

*   快速回查某 SKU / 商品的历史生图结果
*   对比不同模型产出
*   严格遵循数据权限，不返回越权数据
*   点击图片后复用既有 L3 沉浸大屏做只读查看

## 2. 本期范围

v1 做以下能力：

| 能力 | 规则 |
| --- | --- |
| 数据范围 | 仅查询已完成任务及其图片，不包含创建中、生成中、审核中、失败任务 |
| 图片视图 | 按任务分组展示已完成生图，任务下用图片卡片网格展示 |
| 创建人筛选 | 多选树，范围等于当前用户在当前版页面内的数据可见范围 |
| 时间筛选 | 支持创建时间 / 更新时间；按任务时间过滤 |
| 搜索 | 美工版搜系统SKU / TaskID；运营版搜店铺 / ASIN / TaskID；仅精确匹配 |
| 模型筛选 | LinkFox / 自研 / 阿里绘蛙 |
| 图片类型筛选 | 场景图 / 特写图-白底 / 特写图-非白底 / 卖点图 / 标准普通A+图 |

| 分页 | 按任务分页，每页 10 / 20（默认）/ 50 |
| L3 查看 | 点击图片打开既有 L3 沉浸大屏，只读查看 |

## 3. 两版差异

除下表外，美工版与运营版的功能能力、视觉和交互保持一致。

| 维度 | 美工版 | 运营版 |
| --- | --- | --- |
| 页面入口 | 美工版查看已完成生图 | 运营版查看已完成生图 |
| 页面路由 | 美工版独立页面路由 | 运营版独立页面路由 |
| 数据范围 | 美工条线任务/图片 | 运营条线任务/图片 |
| 创建人可选范围 | 按美工版权限规则计算 | 按运营版权限规则计算 |
| 任务身份字段 | 系统SKU | 店铺 + ASIN |
| 搜索范围 | 系统SKU + TaskID | 店铺 + ASIN + TaskID |

两版 v1 都支持 LinkFox / 自研 / 阿里绘蛙、5 种图片类型和侵权禁售检测；这些不再作为版本差异。

## 4. 数据权限

数据权限是本页面的安全核心。后端必须按当前登录用户计算可见范围，并在列表查询时强制裁剪，不能依赖前端筛选保证安全。

权限按以下优先级判定；命中更高优先级时，直接使用更高优先级的可见范围。

**规则 1：系统级管理员直接看全体**

| 版本 / 页面 | 系统级管理员 | 可见范围 |
| --- | --- | --- |
| 运营版 | 深圳运营总监、成都运营总监、本项目产品经理、本项目前后端研发、本项目测试、技术支持 | 运营版全体 |
| 美工版 | 设计总监、所有美工组主管、所有美工子部门高级主管、本项目产品经理、本项目前后端研发、本项目测试、技术支持 | 美工版全体 |

「运营版全体」仅指运营版数据域内的深圳 + 成都运营数据；「美工版全体」仅指美工版数据域内的数据。

如同一登录用户确实需要同时查看美工版和运营版数据，应分别在美工版、运营版权限配置中授予对应页面的数据权限；进入某一版页面后，只能查看该版对应数据库表内的数据，不表示两个页面可以跨数据域混查。

**规则 2：特殊团队 / 特殊人员整体上提可见范围**

| 特殊团队 / 特殊人员 | 上提后的可见范围 |
| --- | --- |
| 深圳业务管理部全体成员、深圳总监助理 | 深圳运营全体，不含成都 |
| 成都 FBA 策略管理组全体成员 | FBA 部门全体 |
| 成都 FBM 策略管理组全体成员 | FBM 部门全体 |

**规则 3：默认按岗位层级看下级**

| 条线 / 岗位 | 默认可见范围 |
| --- | --- |
| 运营专员（深圳 / 成都） | 自己 |
| 运营组主管（深圳 / 成都） | 本组 |
| 运营部门高级主管（深圳 / 成都） | 本部门 |
| 美工专员 | 自己 |

运营条线严格执行组织隔离，组与组、部门与部门之间互不可见；美工条线默认规则为专员看自己，美工组主管、子部门高级主管、设计总监通过规则 1 授予系统级管理员权限。深圳精品、爆旺等扁平部门没有「组」层级：专员看自己，部门高级主管看本部门。

典型例子：

| 登录用户 | 进入页面 | 可见范围 |
| --- | --- | --- |
| 深圳运营专员 | 运营版 | 自己 |
| 深圳 A1 组主管 | 运营版 | A1 组全体 |
| 深圳业务管理部成员 | 运营版 | 深圳运营全体，不含成都 |
| 成都 FBA 策略管理组成员 | 运营版 | FBA 部门全体 |
| 美工专员 | 美工版 | 自己 |
| 美工组主管 | 美工版 | 美工全体 |
| 本项目产品经理 | 美工版 / 运营版 | 当前版页面全体 |

权限计算落地：

*   后端权限计算最终产出当前用户在当前版页面内的「可见创建人集合」。
*   创建人下拉、主列表查询、越权校验都基于这个集合。
*   前端未传创建人筛选条件时，按「可见创建人集合全选」处理。
*   前端传入的创建人必须是「可见创建人集合」的子集；否则返回错误，不返回业务数据。

页面落地：

*   创建人下拉只展示当前用户可见范围内的人，默认全选。
*   用户可取消部分创建人，但至少保留 1 人。
*   精品、爆旺等没有组层级的部门，人员直接挂在部门下展示。
*   美工版和运营版共用公司系统登录账号，但页面路由、权限配置、任务/图片数据彼此独立。

权限验证重点：

*   同一岗位层级在两版规则不同：运营组主管只看本组，美工组主管看美工全体。
*   系统级管理员优先于特殊团队和默认岗位层级，在当前版页面内看全体。
*   特殊团队 / 特殊人员优先于默认岗位层级，例如深圳业务管理部成员即使是专员，也看深圳运营全体。
*   两版数据域必须互相隔离：美工版任务在运营版查不到，运营版任务在美工版查不到。
*   越权条件必须验证：请求不可见创建人或不可见任务时，返回错误，不返回业务数据。

## 5. 页面结构

页面自上而下为：

1. 页头：标题「查看已完成生图」
2. 状态子 Tab + 排序 + 展示密度
3. 筛选区：任务维度筛选、图片维度筛选、查询、重置
4. 主内容：按任务分组的图片卡片网格
5. 分页器

具体视觉和布局以原型为准。

## 6. 功能规则

### 6.1 筛选作用域

| 作用域 | 筛选项 | 行为 |
| --- | --- | --- |
| 任务维度 | 创建人、时间、搜索、审核通过/不通过张数 | 不命中则整个任务从列表移除 |
| 图片维度 | 模型、图片类型、侵权禁售状态、状态子Tab | 不命中则只剔除对应图片 |

如果一个任务下所有图片都被图片维度筛选剔除，该任务也不展示。分页统计的任务数以最终仍有可见图片的任务为准。

查询结果按以下顺序收敛：先应用数据权限，再应用任务维度筛选，再应用图片维度筛选，最后剔除无可见图片的任务并按任务分页。

### 6.2 筛选生效方式

| 项目 | 生效方式 |
| --- | --- |
| 状态子Tab、排序、展示密度 | 即时生效 |
| 创建人、时间、搜索、模型、图片类型、侵权禁售、张数筛选 | 需点击「查询」后生效 |

重置后回到：创建人全选当前可见范围、时间为创建时间最近 7 天、状态子 Tab 为全部图片、页码为第 1 页；排序和展示密度作为视图偏好保留。

### 6.3 关键业务规则

| 功能 | 规则 |
| --- | --- |
| 时间筛选 | 按任务创建时间 / 更新时间过滤，不按单张图片时间过滤 |
| 搜索 | 精确匹配，大小写不敏感，前后空格忽略；不支持模糊、前缀、后缀匹配 |
| 状态子Tab | 「通过 / 不通过」指人工审核状态，与侵权禁售检测状态无关 |
| 状态Tab数字 | 基于当前权限、任务维度筛选、图片维度筛选统计，但不受当前选中的状态Tab影响 |
| 侵权禁售 | 仅卖点图、标准普通A+图进入检测；场景图、特写图-白底、特写图-非白底恒为未检测；筛选项按检测结果状态提供，至少包含全部、未检测、检测通过、侵权风险、禁售风险 |
| 张数筛选 | 仅按任务内审核通过张数 / 审核不通过张数筛选，不做总张数 |
| 排序 | 默认按任务更新时间倒序；用户可切换为任务创建时间 / 更新时间升降序；任务内图片按图片类型顺序和同类型序号展示 |
| 任务统计 | 任务分隔条的总数、通过数、不通过数基于任务真实全量图片，不受当前图片维度筛选影响 |
| 图片序号 | 图片卡片底部序号为任务内同类型图片递增序号，从 1 开始 |
| 分页 | 按任务分页，一个任务下所有可见图片必须在同一页，不跨页拆分 |
| L3 查看 | 点击图片在当前页面上方打开既有 L3 全屏弹窗；关闭后保留原列表位置和筛选条件；本页进入 L3 为只读态，审核、推送、编辑等状态变更操作隐藏或禁用 |

## 7. 边界情况

| 场景 | 期望行为 |
| --- | --- |
| 当前用户无可见数据 | 列表空态；创建人下拉只含自己 |
| 筛选组合无结果 | 列表空态，筛选条件保留 |
| 创建人尝试清空到 0 | 前端阻止，提示「至少保留一位创建人」 |
| 请求越权创建人或越权任务 | 返回错误，不返回业务数据 |
| 图片维度筛选后任务下无可见图片 | 该任务从列表移除 |
